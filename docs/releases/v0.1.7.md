# v0.1.7

## Added

- `ClusterRegistry` for region-to-endpoint registration and health state
- health-aware failover in `NetworkTissueExecutor`
- runnable `TissueNodeServer` with `/health` and `/execute`

## Changed

- network tissue execution now resolves healthy endpoints instead of assuming one fixed node
- remote execution can recover from a failed endpoint by routing to the next healthy candidate
